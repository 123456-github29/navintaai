--------------------------------------------
D) IMPLEMENT REAL LOGIC FOR EACH STEP (MVP BUT REAL)
--------------------------------------------
Each step MUST do something real, even if v1 is “lightweight”.

STEP 1 — Context analysis (REAL)
Inputs:
- video file URL
- transcript (if available) or generate transcript via your existing pipeline
Process:
- Analyze transcript for:
  - pause lengths (use word timestamps)
  - hook intensity (first 2 seconds)
  - sentence boundaries
- Analyze video for:
  - face presence (basic heuristic is OK if no CV model: “assume talking head” unless we have data)
  - brightness/contrast estimate (ffmpeg metadata or basic frame sample)
- Output:
  - a structured “analysis” object saved into step1_summary and/or a JSON column:
    - avg_pause_ms, num_long_pauses, recommended_cut_aggressiveness, brightness_score, etc.

Update edit_runs:
- step1_status running → succeeded
- step1_progress updates as you go

STEP 2 — Storyboard creation (REAL)
This should produce a “storyboard” that maps to an EDL plan (not the final EDL yet).
Process:
- Use the analysis from Step 1 + transcript to create:
  - segment boundaries (clips)
  - which segments need emphasis zoom
  - placeholders for b-roll inserts (optional)
- Output:
  - storyboard JSON:
    - segments: [{startFrame, endFrame, intent: 'hook'|'body'|'cta', zoomTarget}]
  - save as step2_summary and storyboard json (new column or store in edl_json draft)

STEP 3 — Animation (REAL)
Goal: Create animation “instructions” that Remotion will apply.
Process:
- Choose:
  - caption style pack (existing)
  - transition pack (crossfade etc.)
  - overlays (optional: clapperboard or emoji reactions as overlays ONLY if you already have assets; if not, keep as TODO)
- Implement:
  - Convert storyboard → full EDL with:
    - clips array
    - captions track (word-chunking already exists)
    - transitions (4-frame crossfades)
    - camera motions (zoomTarget)
- Output:
  - EDL JSON validated by Zod
  - save to edl_json

STEP 4 — Integration (FINAL RENDER + EXPORT)
Process:
- Call RendererService to render EDL into mp4
- Upload mp4 to storage and produce signed URL
- Update edit_runs with output_url
- Mark step4_status succeeded and status succeeded

If any step fails:
- mark that step failed, status failed, store error message
- UI shows retry button “Retry from Step X”
