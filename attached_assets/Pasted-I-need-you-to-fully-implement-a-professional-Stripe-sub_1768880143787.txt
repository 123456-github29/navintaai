REQUIRED IMPLEMENTATION (DO ALL STEPS)

1. Install Stripe

Install Stripe SDK on backend:
	•	stripe

⸻

2. Environment Variables (read from secrets)

Use these env vars (assume they exist in Replit Secrets):
	•	STRIPE_SECRET_KEY
	•	STRIPE_WEBHOOK_SECRET
	•	STRIPE_PRICE_ID
	•	PUBLIC_URL (https://navinta.org)

DO NOT hardcode keys.

⸻

3. Database (Supabase / Postgres)

Create or migrate a table:

user_subscriptions
	•	user_id (uuid, primary key, references auth.users.id)
	•	stripe_customer_id (text)
	•	stripe_subscription_id (text)
	•	status (text)
	•	current_period_end (timestamptz)
	•	created_at (timestamptz default now())

⸻

4. Backend – Billing Routes (Express)

Create these routes:

POST /api/billing/create-checkout-session
	•	Requires authenticated user
	•	Creates Stripe Checkout Session
	•	mode: subscription
	•	price: STRIPE_PRICE_ID
	•	success_url = ${PUBLIC_URL}/billing/success
	•	cancel_url = ${PUBLIC_URL}/billing/cancel
	•	Attach user id in metadata.userId
	•	Return { url }

⸻

POST /api/billing/create-portal-session
	•	Requires authenticated user
	•	Fetch stripe_customer_id from DB
	•	Create Stripe customer portal session
	•	Return { url }

⸻

POST /api/billing/webhook
	•	MUST use raw body
	•	Verify signature with STRIPE_WEBHOOK_SECRET
	•	Handle events:
	•	checkout.session.completed
	•	customer.subscription.updated
	•	customer.subscription.deleted

For these:
	•	Insert or update user_subscriptions
	•	Keep status in sync
	•	Update current_period_end
	•	Never trust frontend state

⸻

5. Backend – Paywall Enforcement

Create middleware:
requireActiveSubscription
