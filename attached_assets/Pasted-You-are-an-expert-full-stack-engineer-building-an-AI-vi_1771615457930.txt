You are an expert full-stack engineer building an AI video editor. Implement a “B-roll Slot Filler” system that ONLY generates/retrieves B-roll assets, while the Director/Editor systems control timing and placement. This must be production-quality, schema-validated, and source-adapter based.

SCOPE (IMPORTANT)
- Only implement the B-roll system that fills “B-roll slots” with candidate clips + suggested trims.
- Do NOT implement directing, timeline placement, captioning, or full rendering.
- The Director will provide slots with start/end times and constraints; our job is ONLY to return the best B-roll candidates for each slot.

CORE IDEA
- Director outputs BrollSlot[] (with intent + constraints + duration target).
- Broll system calls one or more stock video APIs (start with Pexels Video API) to retrieve candidate clips for each slot.
- Broll system ranks candidates by relevance + duration fit + style constraints + novelty rules.
- Output: For each slot, return top K candidates with score and suggested in/out points (trim inside the candidate clip). The Editor will place them at the slot times.

TECH STACK ASSUMPTIONS
- Node.js + TypeScript backend (or JS allowed if repo is JS)
- Express or Next.js API routes (detect project; implement accordingly)
- Use Zod for schema validation
- Use fetch/axios for HTTP calls
- Use environment variables for API keys:
  - PEXELS_API_KEY

3) Ranking + Scoring Engine
Create: src/systems/broll/scoring.ts

Implement:
- normalizeTokens(str): lowercases, trims, removes punctuation
- token overlap score between intent.keywords and clip tags (if tags missing, use query tokens vs clip url text)
- entityScore:
  - if any entity token appears in tags/title/url -> boost
  - else 0
- durationFit:
  - compute how well clip.durationMs can cover durationTargetMs
  - ideal: clip.durationMs >= target and not excessively longer (cap)
- cropFit:
  - if cropMode is 9:16-safe, prefer clips with:
     - width/height close to 9:16, OR
     - high resolution allowing crop (width >= 1080 and height >= 1920 OR width >= 1920 and height >= 1080)
  - output 0..1
- styleFit (MVP heuristic):
  - preferTags/avoidTags influence
  - visualEnergy influences:
     - dynamic: prefer tags like ["action", "sports", "movement", "fast"] (if tags absent, skip)
     - cinematic: prefer tags like ["cinematic", "sunset", "moody"] (if tags absent, skip)
  - if constraints are "any", styleFit defaults to 0.5
- semanticScore (MVP):
  - if no embeddings, set semanticScore = 0.55 baseline
  - (optional) if project already has embeddings infra, implement cosine similarity

Combine into overallScore:
- overall = 0.35*semantic + 0.20*keyword + 0.20*durationFit + 0.15*cropFit + 0.10*styleFit
- entityScore acts as a multiplier boost: overall *= (1 + 0.25*entityScore) capped at 1

Add global quality gate:
- keep candidates with overallScore >= 0.60
- always return at least 1 candidate if any exist (even if below threshold), but mark reason “low confidence”

Add novelty / dedup rules:
- avoid selecting the same assetId for multiple slots unless necessary
- respect globalRules.minGapMsBetweenSameAsset by tracking used assetIds and slot start times
Return topK per slot.

4) Suggested Trim Logic

- suggestedTrim.outMs = inMs + target (clamped)
